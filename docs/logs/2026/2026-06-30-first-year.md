# Solvinter Edge — First Year

## Summary

During the first year, Solvinter moved from idea to practical experimentation. The work has not yet produced a finished company, but it has produced infrastructure, learning, tools, hardware, field experience and a clearer direction.

## Skills learned

- Terminal basics
- Git and GitHub
- VS Code
- Python basics
- Next.js basics
- Markdown
- YAML
- Static website deployment
- AI-assisted development

## Hardware and compute

- Built and tested old PCs
- Tested GPU compute
- Tested small language models
- Tested cryptocurrency mining
- Tested Storj and distributed storage
- Acquired Raspberry Pi units
- Acquired Dell server with space for eight hard drives
- Used GPUs for distributed scientific compute when not used for mining

## Ghana infrastructure

- Built a solar UPS prototype in Ghana
- Used solar panel, MPPT, two car batteries, charger and battery shunt
- Designed the system to support Starlink
- Explored remote monitoring and resilient connectivity
- Identified Cow Lane as first urban workspace node

## Networks and access

- Tested Tailscale
- Explored remote administration
- Installed and tested Starlink-related infrastructure
- Investigated how digital work could be performed from Ghana

## AI work and markets

- Created Upwork account
- Registered with Invisible / Meridial
- Investigated AI training, AI evaluation and remote digital work
- Began studying how human expertise becomes part of AI production

## Physical assets

- Existing house near the Atlantic coast in Ghana
- Family space in Cow Lane
- Hardware acquired for experimentation
- Solvinter Ltd registered

## Core observation

If one person can create value with a computer, internet access and AI tools, then people in Ghana may also be able to do so if given infrastructure, training and access to markets.

## Open questions

- How can this be done ethically?
- How should Cow Lane be designed as a human workspace?
- What can be learned before scaling to other nodes?
